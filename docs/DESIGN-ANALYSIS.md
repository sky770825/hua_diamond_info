# 成員廣告牆｜設計分析與改善建議

本文針對**成員牆**的視覺、資訊架構、互動、可及性與內容**逐一分析**，並列出**可加強與可新增的具體建議**，便於排程實作。

---

## 一、目前設計整理

### 1.1 設計系統

| 項目 | 現況 |
|------|------|
| **主題** | 深色科技風（背景 ~6%、secondary ~17%、primary 青藍） |
| **字體** | Inter、Noto Sans TC、system-ui |
| **圓角** | `--radius: 0.75rem`，卡片 / 按鈕 / 輸入框一致 |
| **陰影** | 卡片有 primary 色 glow、hover 加深 |
| **組件** | glass-card、text-gradient、touch-target、line-clamp |

### 1.2 版面結構

- **Header**：標題「資訊組｜成員牆」、副標、搜尋框、清除篩選、重新整理
- **Filter**：技能標籤（全部 + 動態 tags），可多選
- **Main**：成員卡片網格（1→2→3→4 欄 RWD）
- **Footer**：成員數、篩選摘要、最後更新日
- **Detail**：彈窗（形象照、NO/姓名、引薦需求三層、服務項目、作品集）

### 1.3 成員卡片

- 頂部 4:3 形象照橫幅（無圖則縮寫 + 漸層）
- NO.、姓名、引薦需求摘要（line-clamp-2）、服務項目前 3 項、tags、作品數
- Hover 上浮、陰影強化、形象照微放大；「點擊查看完整資訊」提示

### 1.4 已具備的優點

- 深色主題統一、primary 點綴清楚
- 搜尋 + 標籤篩選實用；RWD、safe-area、觸控目標已考量
- 形象照、引薦需求、服務項目、作品集層次清楚；彈窗可捲動

## 二、可加強／可新增的建議（依類別）

### 2.1 視覺與品牌

| # | 建議 | 說明 | 優先度 |
|---|------|------|--------|
| V1 | **Logo / 識別圖** | Header 加上資訊組 logo 或小 icon，強化品牌辨識 | 中 |
| V2 | **淺色主題選項** | 支援 `prefers-color-scheme: light` 或手動切換，照顧不同環境與偏好 | 中 |
| V3 | **標題區視覺強化** | 副標下可加 subtle 裝飾線、或小型 CTA（例：「如何加入」） | 低 |
| V4 | **卡片 hover 狀態區分** | 有無形象照、有無作品可略做視覺區隔（如細框、badge 樣式） | 低 |

### 2.2 資訊架構與層次

| # | 建議 | 說明 | 優先度 |
|---|------|------|--------|
| I1 | **「聯絡」或「進一步了解」CTA** | 詳情彈窗加明顯按鈕（LINE、Email、表單連結等），方便轉換成實際接觸 | 高 |
| I2 | **聯絡方式資料與 API** | Member 新增 `contact?: { line?, email?, phone? }`，後台可編輯、前端唯讀顯示 | 高 |
| I3 | **卡片上標籤與篩選聯動** | 點卡片內 tag 可套用該標籤篩選（或預填搜尋），減少重複操作 | 中 |
| I4 | **「最後更新」意義** | 若為資料異動時間，改為「資料更新」並由 API 提供；若為前端 build 時間則註明 | 低 |
| I5 | **排序選項** | 支援依姓名、編號、作品數等排序，滿足「找最新 / 最活躍」等情境 | 中 |

### 2.3 互動與回饋

| # | 建議 | 說明 | 優先度 |
|---|------|------|--------|
| U1 | **搜尋防抖（debounce）** | 輸入時延遲 200–300ms 再過濾，減少輸入中閃爍與運算 | 中 |
| U2 | **篩選結果數量即時預覽** | 在標籤旁顯示「選此 tag 後約 N 位成員」，輔助決策 | 低 |
| U3 | **鍵盤與焦點** | 彈窗內 Tab 順序、Esc 關閉、關閉鈕 `:focus-visible` 樣式確保鍵盤可及 | 中 |
| U4 | **卡片 loading / skeleton** | 列表載入時以骨架卡取代全版 spinner，感覺更快、版型不跳動 | 中 |
| U5 | **作品集圖片 lazy load** | 詳情內作品圖 `loading="lazy"`，卷動進視窗再載入 | 中 |

### 2.4 可及性（a11y）

| # | 建議 | 說明 | 優先度 |
|---|------|------|--------|
| A1 | **焦點指示與 skip link** | 全站 `:focus-visible` 明顯、必要時加「跳到主內容」連結 | 中 |
| A2 | **語意與 ARIA** | 篩選區 `role="search"`、標籤 `role="group"`、結果區 `aria-live`  announcements | 中 |
| A3 | **彈窗 `aria-modal`、`aria-labelledby`** | 確認 Radix Dialog 已正確設定，螢幕報讀能識別為彈窗與標題 | 中 |
| A4 | **色彩對比** | 檢查 muted-foreground、placeholder、badge 在深色底上的對比比（建議 ≥4.5:1） | 中 |
| A5 | **圖片 `alt`** | 作品集 `alt` 依 `title` 或 `description` 組成，無則 fallback 如「成員 OOO 的作品」 | 低 |

### 2.5 內容與資料

| # | 建議 | 說明 | 優先度 |
|---|------|------|--------|
| C1 | **成員簡短一句話（headline）** | 若 Member 有 `headline`，卡片與詳情可優先顯示，方便快速掃描 | 中 |
| C2 | **作品集連結** | 作品若有外連（如 Behance、個人站），可加 `url` 欄位與「前往」按鈕 | 低 |
| C3 | **標籤分組或分類** | 若標籤多，可依「技能／產業／服務類型」分組顯示，便於瀏覽 | 低 |

### 2.6 效能與技術

| # | 建議 | 說明 | 優先度 |
|---|------|------|--------|
| P1 | **形象照與作品圖 responsive** | 後端或 CDN 提供多尺寸／WebP；前端 `srcset` / `<picture>`，減少大圖下載 | 中 |
| P2 | **列表虛擬化** | 成員數 > 50 時考量虛擬捲動，維持流暢 | 低 |
| P3 | **API 快取 / 重新驗證** | 前端對 GET /api/members 做短期 cache（如 SWR、React Query staleTime），減少重複請求 | 低 |

### 2.7 空狀態與邊界

| # | 建議 | 說明 | 優先度 |
|---|------|------|--------|
| E1 | **無成員時的說明** | 若 `members.length === 0`，顯示「尚無成員」與簡短說明或後台連結，避免空白困惑 | 中 |
| E2 | **無搜尋結果的建議** | 現有「找不到符合條件」已具備；可加「試試看放寬標籤」等短句 | 低 |
| E3 | **作品集為空** | 詳情內已處理；若未來有「邀請上傳」等 CTA 可再補 | 低 |

### 2.8 其他功能擴充

| # | 建議 | 說明 | 優先度 |
|---|------|------|--------|
| F1 | **分享單一成員** | 詳情頁「分享」複製連結（如 `/?member=016`），便於轉傳 | 中 |
| F2 | **SEO / 共用預覽** | 依成員動態生成 `og:image`、`og:title`、`description`，分享到社群時更完整 | 低 |
| F3 | **列印友善** | 列印時隱藏篩選、按鈕，保留標題 + 卡片列表，便於紙本參考 | 低 |

## 三、建議實作順序（依效益／成本）

### 高優先（建議先做）

1. **I1 + I2**：聯絡 CTA 與聯絡方式資料／API，直接提升「找到人→聯絡」轉換。
2. **U4**：卡片 skeleton 取代全版 loading，改善體感速度與版面穩定。
3. **E1**：無成員時的空狀態說明，避免空白頁。

### 中優先（有餘力再做）

4. **U1**：搜尋 debounce。  
5. **U5**：作品集圖片 `loading="lazy"`。  
6. **I3**：卡片標籤點擊套用篩選。  
7. **I5**：排序選項。  
8. **A1–A4**：焦點、ARIA、對比等 a11y 補強。  
9. **V2**：淺色主題選項（若使用者有需求）。  
10. **F1**：分享單一成員連結。

### 低優先（可視需求排程）

11. **V1, V3, V4**：Logo、標題裝飾、卡片細微區分。  
12. **I4, U2, C1–C3, P2–P3, E2–E3, F2–F3**：其餘項目。

---

## 四、小結

- **已有基礎**：視覺統一、RWD、觸控、基本篩選與詳情流程都到位。  
- **最值得補**：**聯絡 CTA + 聯絡資料**、**列表 loading 改成 skeleton**、**無成員空狀態**。  
- **再來**：搜尋 debounce、lazy load、標籤聯動、排序、a11y、淺色主題、分享連結等，可依資源與目標逐步做。

若你希望先實作某一項（例如 I1+I2、U4 或 E1），可指定編號，我可以依現有架構寫出具體修改方案與程式位置。
